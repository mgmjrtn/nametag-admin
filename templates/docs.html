Nametag PoC Workflow Tool — Functional Summary

A workflow tool powered by Agent AI that helps Nametag reduce time spent on repetitive operational work, navigate disconnected systems more quickly, and make it easier to work with data that may not match across sources.

Purpose

The tool is built around a simple idea: let a user interact in plain language through chat, route the request to the right workflow, add identity verification when needed, and return a useful output quickly in a format the user can act on.

In this Proof of Concept, the tool demonstrates four live functions:

  • Quick Docs Helper
  • Customer Data Analysis Agent
  • Operations notes summary/email
  • Operations quick time entry

What problem this solves

The demo is built around the operating challenges described in the Nametag scenario:

  • Disconnected systems
  • Manual workflows when exceptions arise
  • Inconsistent or unreliable data across tools
  • Rising audit/compliance pressure
  • Engineers spending time on operational work instead of product work
  • Pressure to simplify, not add more tooling

This workflow tool addresses those challenges by giving users a single conversational entry point, reducing context switching, speeding up common operational tasks, and creating outputs that are easier to review, share, and act on.

Available commands

Function 1: Quick Docs Helper

Helps a user find and work with internal documentation more quickly. Designed for moments when someone needs a fast answer, a summary, or help locating the right information without manually searching through multiple sources.

How it works: The user asks in plain language → the workflow recognizes it as a docs request → the agent retrieves relevant content → returns a concise, readable answer through chat.

Business value: Reduces time lost searching for information, lowers friction for common internal questions, and helps teams navigate knowledge faster without extra manual effort.

Function 2: Customer Data Analysis Agent

Helps a user review customer-related data when that data exists in multiple sources and does not fully match. Especially useful when someone needs to understand what is happening across data sources and where discrepancies may exist.

How it works: The user requests a data lookup → the workflow routes to the data function → the agent reads from configured sources → compares values and identifies meaningful differences → returns a summarized analysis.

Crawl → Walk → Run framing:

  • Crawl: Recognize the data problem exists
  • Walk: Compare sources, explain mismatches, help users act faster
  • Run: Eventually integrate more deeply, merge sources, or update system-of-record design

Function 3: Operations notes summary/email

Turns meeting notes or operational notes into a concise summary and an email-ready output. Designed to reduce the follow-up burden that often falls on engineering or operations staff after meetings.

How it works: The user submits notes → the workflow routes to the operations notes function → the agent summarizes → prepares or sends the email output → result is returned through chat and email.

Business value: Reduces repetitive admin work after meetings, improves consistency of summaries, and helps engineers spend less time on operational documentation.

Function 4: Operations quick time entry

Helps a user create a structured time-entry output quickly through chat. In the current demo, the output is a CSV delivered back through WhatsApp.

How it works: The user sends time entry information through chat → the workflow routes to the time-entry function → the agent structures the entries into the required CSV format → the output is returned through WhatsApp.

Business value: Reduces a manual operational task that can consume about an hour each week, gives users a faster path from raw work notes to a structured upload format.

Identity verification in the workflow

For protected functions (operations admin and data find), the workflow includes a Nametag-style identity verification sequence.

Verification sequence: Request begins → verification is initiated → a link/session is completed → the verified result returns into the workflow → the workflow continues with the downstream action.

This shows where Nametag fits in a broader operational flow, rather than presenting identity verification as a standalone feature.

Demo assumptions

  • Customer performance data exists in multiple sources and does not fully match
  • Engineering teams are spending too much time on operational work
  • Two especially valuable operational targets: meeting notes summaries/emails, and weekly timecard entry
  • The customer wants simplification, not another complicated tool

Nametag Demo Admin — User Guide

A lightweight configuration manager for the Nametag n8n demo environment. Manage the Google Sheets that drive the demo workflows through a clean browser-based interface — without ever opening a spreadsheet directly.

URL: https://nametag-admin.onrender.com

What this tool manages

SectionWhat it controls
Nametag Data Configuration The list of data sources (name + link) used by the demo
Nametag Timecard CSV Configuration The field definitions for the timecard CSV output format

Changes made here are written directly to the underlying Google Sheets in real time. The n8n demo workflow reads from those same sheets, so changes take effect immediately.

Getting started

  1. Open a browser and go to https://nametag-admin.onrender.com
  2. You'll land on the Dashboard — two cards, one for each configuration area
  3. Click Open → on either card to enter that section

⚠️ The app runs on Render's free tier and will sleep after 15 minutes of inactivity. If the page takes 20–30 seconds to load, that's normal — it's waking up. Open it a minute before you need it during a demo.

Nametag Data Configuration

Manages the list of named data sources used by the demo. Each entry has a Source Name and a Link (a Google Sheets URL). The demo workflow uses this list to know which data sources are available.

Add a new entry

  1. Scroll to the Add New Source form at the bottom of the page
  2. Enter a Source Name (required) and Link (optional but recommended)
  3. Click Add

Edit an existing entry

  1. Click Edit on the row you want to change
  2. An inline form will appear with current values pre-filled
  3. Make your changes and click Save, or Cancel to discard

Delete an entry

  1. Click Delete on the row you want to remove
  2. Confirm the deletion when prompted

⚠️ Deleting an entry removes it from the Google Sheet permanently. There is no undo.

Nametag Timecard CSV Configuration

Manages the field definitions that control the format of the timecard CSV output. Each entry defines a field name (the column header) and a description of the expected data format.

Add a new field

  1. Scroll to the Add New Field form at the bottom
  2. Enter the field name (required) and a description of valid values
  3. Click Add

Edit an existing field

  1. Click Edit on the row you want to update
  2. Modify the field name or description
  3. Click Save or Cancel

Delete a field

  1. Click Delete on the row to remove
  2. Confirm when prompted

⚠️ Make sure the demo workflow doesn't depend on a field before deleting it.

Tips & notes

  • Changes are live immediately. No publish or sync step — changes write to Google Sheets right away.
  • Row numbers matter. Be cautious about deleting entries mid-list, as it shifts row numbers for everything below.
  • The header row is protected. Row 1 in each sheet is never shown or editable in this app.
  • First load may be slow. Plan to open the app a minute before you need it during a demo.

Troubleshooting

ProblemWhat to check
"Could not read sheet" error The Google Sheet may no longer be shared publicly. Open the sheet, click Share, confirm "Anyone with the link" can view.
"Write failed" error when saving Paste the Apps Script URL in a browser — it should return {"status":"ok"}. Also confirm "Who has access" is set to "Anyone" in the Apps Script deployment.
Page is blank or won't load The app may be sleeping. Wait 30–60 seconds and refresh.
Changes don't show up in the demo Re-trigger the n8n workflow after making changes — it reads the sheet at trigger time.
